这一部分主要聊一下， rust中的生命周期

&’static

&'static对于生命周期有着非常强的要求：一个引用必须要活得跟剩下得程序一样就，才能被标注为&'static。

对于字符串字面量来说，它直接被打包到二进制文件中，永远不会被drop，因此它能跟程序活得一样久，自然它得生命周期是'static。

但是, &'static生命周期针对得仅仅是引用，而不是持有该应用得变量，对于变量来说，还是要遵循相应得作用域规则。

實驗環境部署: Docker

出於對操作複雜度的考慮，個人考慮使用Docker進行實驗環境的部署，從官方的文檔來看，只需要兩個命令make build_docker和make docker就可以完成實驗的部署。

因爲官方提供的Docker的基礎鏡像是Ubuntu:20.04，如果使用默認的官方鏡像，則在網絡不好的情況下，apt-get這個速度會十分緩慢，因此需要對基礎的Ubuntu:20.04鏡像做一些修改：

首先，測試了一下，基礎的Ubuntu:20.04中存在sed命令，可以通過這個命令，修改/etc/apt/sources.list中的鏡像源的列表，爲了簡化操作的複雜度，我添加了一些文件為基礎的文件

在registryList中輸入相應的Ubuntu 20.04的相關鏡像源。

關於如何參與開源項目

這裏僅供參考

首先，找到對應的issue -> fork官方的倉庫 -> 把自己的fork下來的倉庫clone到相應的開發環境 -> 一般開源提交，先創建一個branch，命名方式推薦 issue-{issue_id} -> 將本地的代碼提交到雲端的倉庫中，並創建相應的 issue-{issu_id} branch -> 解決與上游代碼的衝突， 提交pull request
