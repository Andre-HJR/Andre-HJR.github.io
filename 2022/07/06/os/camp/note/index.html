實驗環境部署: Docker

處於對操作複雜度的考慮，個人考慮使用Docker進行實驗環境的部署，從官方的文檔來看，只需要兩個命令make build_docker和make docker就可以完成實驗的部署。

因爲官方提供的Docker的基礎鏡像是Ubuntu:20.04，如果使用默認的官方鏡像，則在網絡不好的情況下，apt-get這個速度會十分緩慢，因此需要對基礎的Ubuntu:20.04鏡像做一些修改：

關於如何參與開源項目

這裏僅供參考

首先，找到對應的issue -> fork官方的倉庫 -> 把自己的fork下來的倉庫clone到相應的開發環境 -> 一般開源提交，先創建一個branch，命名方式推薦 issue-{issue_id} -> 將本地的代碼提交到雲端的倉庫中，並創建相應的 issue-{issu_id} branch -> 解決與上游代碼的衝突， 提交pull request
